تعليمات التطوير لتطبيق tageep
=============================

هذا الملف مخصص للمطور المبتدئ الذي يريد تعديل تطبيق tageep داخل Frappe Bench.


1. مكان الملفات المهمة
---------------------

جذر التطبيق:
apps/tageep

ملفات بايثون الخاصة بالتطبيق:
apps/tageep/tageep

ملف hooks الذي تربط فيه المسارات والإعدادات:
apps/tageep/tageep/hooks.py

ملف API الذي يحتوي دوال الباكند:
apps/tageep/tageep/api.py

صفحات الويب:
apps/tageep/tageep/www

ملفات CSS:
apps/tageep/tageep/public/css

ملفات JavaScript:
apps/tageep/tageep/public/js

رابط الأصول في المتصفح يكون بهذا الشكل:
/assets/tageep/css/tageep.css
/assets/tageep/js/tageep.js


2. تشغيل النظام
---------------

من داخل مجلد bench:

cd /home/newsmart/frappe-bench2
bench start

رابط صفحة tageep:

http://192.168.0.248:8001/tageep

الموقع الافتراضي الذي يخدم رابط IP هو site1.local.


3. أوامر مفيدة للتحقق
---------------------

فحص أن التطبيق مثبت على الموقع:

bench --site site1.local list-apps

مسح الكاش بعد تعديل hooks أو الصفحات:

bench --site site1.local clear-cache

تشغيل migrations بعد تعديل الجداول أو patches:

bench --site site1.local migrate

فحص أخطاء بايثون:

python3 -m py_compile apps/tageep/tageep/api.py apps/tageep/tageep/hooks.py

فحص أخطاء JavaScript:

node --check apps/tageep/tageep/public/js/tageep.js

اختبار أن صفحة tageep تعمل:

curl -I http://127.0.0.1:8001/tageep

اختبار API الترخيص:

curl http://127.0.0.1:8001/api/method/tageep.api.license_status

اختبار تسجيل الدخول:

curl -X POST \
  -H 'Content-Type: application/json' \
  --data '{"username":"admin","password":"admin"}' \
  http://127.0.0.1:8001/api/method/tageep.api.login


4. كيف تحل الأخطاء
------------------

أولا اقرأ نص الخطأ كاملًا، ولا تكتف بالرسالة الظاهرة في المتصفح.

أماكن السجلات:

logs/frappe.log
logs/bench.log
logs/worker.log
logs/worker.error.log

مثال:

tail -80 logs/frappe.log

إذا ظهر خطأ AuthenticationError:

1. تأكد أن تسجيل الدخول نفسه يعمل:

curl -X POST \
  -H 'Content-Type: application/json' \
  --data '{"username":"admin","password":"admin"}' \
  http://127.0.0.1:8001/api/method/tageep.api.login

2. إذا رجع access_token، اختبر تحميل البيانات:

curl \
  -H 'X-Tageep-Token: ضع_التوكن_هنا' \
  http://127.0.0.1:8001/api/method/tageep.api.get_state

ملاحظة مهمة:
لا تستخدم Authorization لتوكن tageep، لأن Frappe قد يتعامل معه كتوكن API خاص به.
استخدم X-Tageep-Token.

إذا ظهر خطأ 404 عند فتح صفحة:

1. تأكد أن ملف الصفحة موجود داخل:
apps/tageep/tageep/www

2. تأكد أن route موجود في hooks.py:

website_route_rules = [
    {"from_route": "/tageep", "to_route": "tageep"},
]

3. امسح الكاش:

bench --site site1.local clear-cache

إذا لم تظهر تعديلات JavaScript أو CSS:

1. زد رقم النسخة في رابط الملف داخل HTML، مثال:

<script src="/assets/tageep/js/tageep.js?v=20260608-4"></script>

2. اضغط Ctrl + F5 في المتصفح.


5. كيف تنشئ واجهة جديدة وتربطها
-------------------------------

مثال: نريد صفحة جديدة اسمها reports ورابطها:

/tageep-reports

الخطوة 1: أنشئ ملف HTML:

apps/tageep/tageep/www/tageep_reports.html

مثال بسيط:

<!DOCTYPE html>
<html lang="ar" dir="rtl">
<head>
    <meta charset="UTF-8">
    <title>تقارير التعقيب</title>
    <link rel="stylesheet" href="/assets/tageep/css/tageep_reports.css">
</head>
<body>
    <h1>تقارير التعقيب</h1>
    <button onclick="loadReports()">تحميل التقارير</button>
    <pre id="result"></pre>
    <script src="/assets/tageep/js/tageep_reports.js?v=1"></script>
</body>
</html>

الخطوة 2: أنشئ CSS:

apps/tageep/tageep/public/css/tageep_reports.css

مثال:

body {
    font-family: Tahoma, sans-serif;
    margin: 24px;
}

الخطوة 3: أنشئ JavaScript:

apps/tageep/tageep/public/js/tageep_reports.js

مثال:

async function loadReports() {
    const response = await fetch('/api/method/tageep.api.license_status');
    const data = await response.json();
    document.getElementById('result').textContent = JSON.stringify(data, null, 2);
}

الخطوة 4: اربط المسار في hooks.py:

website_route_rules = [
    {"from_route": "/tageep", "to_route": "tageep"},
    {"from_route": "/tageep-reports", "to_route": "tageep_reports"},
]

الخطوة 5: امسح الكاش وجرب:

bench --site site1.local clear-cache
curl -I http://127.0.0.1:8001/tageep-reports


6. كيف تضيف API جديد
--------------------

افتح:

apps/tageep/tageep/api.py

أضف دالة:

@frappe.whitelist(allow_guest=True, methods=["GET"])
def ping():
    return {"ok": True, "message": "pong"}

ثم اختبر:

curl http://127.0.0.1:8001/api/method/tageep.api.ping

إذا كانت الدالة تحتاج تسجيل دخول tageep، استخدم:

@frappe.whitelist(allow_guest=True, methods=["GET"])
def secure_ping():
    user = _current_tageep_user()
    return {"ok": True, "user": user.get("name")}

واختبرها بالتوكن:

curl \
  -H 'X-Tageep-Token: ضع_التوكن_هنا' \
  http://127.0.0.1:8001/api/method/tageep.api.secure_ping


7. نصائح قبل حفظ أي تعديل
-------------------------

1. اقرأ الملف الذي ستعدله قبل التعديل.
2. عدل أقل قدر ممكن.
3. بعد تعديل Python شغل py_compile.
4. بعد تعديل JavaScript شغل node --check.
5. بعد تعديل hooks.py شغل clear-cache.
6. بعد تعديل الجداول أو patches شغل migrate.
7. اختبر الصفحة من المتصفح ومن curl.
8. لا تحذف ملفات أو تغييرات قديمة إلا إذا كنت متأكدًا أنها غير مستخدمة.


8. بيانات الدخول الافتراضية
---------------------------

مستخدم tageep الافتراضي:

admin

كلمة المرور الافتراضية:

admin

هذه ليست بيانات دخول Frappe Desk، بل بيانات دخول صفحة tageep فقط.
